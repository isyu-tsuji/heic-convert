# 広告導入とSEOキーワード設計: HEIC変換サイト

## 目的

- 広告を入れつつ、サイトの主訴求である「画像をアップロードしない」「サーバ送信なし」「ブラウザ内処理」を崩さない。
- 検索流入の入口を、利用者が実際に打ちそうな語から逆算して作る。
- タイトルと説明文は、競合のように英語ブランド先行ではなく、検索語先行で組む。

## 広告導入の手順

1. 先に運営面を整える。
   - プライバシーポリシーを作る。
   - お問い合わせまたは運営者情報を置く。
   - 「画像データはサーバへ送信しない」ことと、「広告配信のために広告事業者へ通信する場合がある」ことを分けて説明する。
2. 広告枠の場所を決める。
   - 変換操作の中心を邪魔しない位置に置く。
   - トップページならサイドバー下部か、ガイド群の下が候補。
   - 記事ページなら本文の上下に少数配置する。
3. 実装方針を決める。
   - 最初は AdSense を想定する。
   - 変換画面の主要操作に広告を重ねない。
   - クリック誤誘導を避ける。
4. CSP と配信を確認する。
   - 外部広告スクリプトが必要になるため、CSP を別途調整する。
   - 画像変換のWASM系CSPと広告用CSPは分けて考える。
5. 公開前に確認する。
   - モバイル幅で広告が崩れないか。
   - 変換ボタン、ドラッグ&ドロップ、結果一覧の視認性を壊していないか。
   - 広告非表示時でもレイアウトが破綻しないか。

## 説明文の方針

- 1文目で「サーバ送信なし」「ブラウザ内処理」を明示する。
- 2文目で「複数ファイル」「JPG/PNG/WebP」を入れる。
- 「安全」「万全」は断定が強いので、事実ベースの語に置き換える。

## 記事設計の方針

- `iPhone` 系は、写真をそのまま配る人向けの検索意図に合わせる。
- `Windows` 系は、「開けない」「見られない」「添付で困る」文脈に寄せる。
- `HEICとは` は、形式の説明だけでなく、なぜ変換が必要かを先に置く。
- `FAQ` は、送信しないことを明確に書いて信頼を取りにいく。

## メモ

- ユーザは `iphone` だけでなく `iPhone`、`heic`、`HEIC`、`写真 変換` みたいなゆれで探す。
- タイトルは検索語を前に寄せ、ブランド名は後ろに置くほうが拾いやすい。
- 広告は後から足せるが、信頼訴求を損なう配置は先に避ける。
