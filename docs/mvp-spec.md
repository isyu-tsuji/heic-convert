# HEICブラウザ変換サイト MVP仕様

## 目的

ブラウザだけでHEIC/HEIF画像をJPG、PNG、WebPへ変換できる最小構成のWebアプリを作る。
画像ファイルはサーバへ送信せず、デコード、エンコード、ZIP生成、ダウンロードまでをすべてクライアント内で完結させる。

## MVPスコープ

- HEIC/HEIFファイルを複数選択できる。
- ドラッグ&ドロップでファイルを追加できる。
- 出力形式をJPG、PNG、WebPから1つ選べる。
- 追加済みファイルを一覧で確認できる。
- 変換状態をファイル単位で確認できる。
- 変換済みファイルをZIPで一括ダウンロードできる。
- 変換処理は`libheif-js`のWASMを使ってブラウザ内で行う。

## MVP対象外

- サーバアップロード、クラウド保存、外部API連携。
- 認証、課金、アナリティクス。
- EXIFやメタデータの完全維持。
- 個別ファイルの単体ダウンロード。
- 画像編集、リサイズ、回転補正の手動指定。
- HEIC以外の入力形式の変換。

## 入力仕様

受け付けるファイル:

- 拡張子: `.heic`, `.heif`
- MIMEタイプ: `image/heic`, `image/heif`

入力バリデーション:

- ファイル選択とドロップ時にZodで境界バリデーションを行う。
- 有効なファイルは`HeicFile` branded typeとして扱う。
- 無効なファイルは一覧へ追加せず、画面上に理由を表示する。
- MVPではファイルサイズ上限は固定しない。メモリ不足やデコード失敗は変換エラーとして扱う。

重複ファイル:

- 同名ファイルも追加できる。
- ZIP内の出力名が重複する場合は`name-1.jpg`のように連番を付ける。

## 出力仕様

対応形式:

- JPG: MIMEタイプ`image/jpeg`、拡張子`.jpg`
- PNG: MIMEタイプ`image/png`、拡張子`.png`
- WebP: MIMEタイプ`image/webp`、拡張子`.webp`

品質:

- JPGとWebPはCanvasの`toBlob`へ品質`0.92`を指定する。
- PNGはブラウザ実装に従い、品質値は指定しない。

出力ファイル名:

- 入力ファイル名から`.heic`または`.heif`を取り除き、選択形式の拡張子を付ける。
- 例: `photo.heic`をJPG変換した場合は`photo.jpg`。

ZIP:

- ZIPファイル名は`converted-images.zip`。
- ZIP生成には`jszip`を使う。
- ZIPには変換成功したファイルのみを含める。
- すべての変換が失敗した場合、ZIPダウンロードはできない。

## 画面仕様

初期画面:

- ヘッダーにアプリ名と「ブラウザ内で完結する」旨を短く表示する。
- ドラッグ&ドロップ領域を表示する。
- ファイル選択ボタンを表示する。
- 出力形式の選択UIを表示する。
- ファイル未追加時は変換ボタンを無効にする。

ファイル追加後:

- ファイル名、サイズ、状態を一覧表示する。
- 状態は`待機中`、`変換中`、`完了`、`失敗`のいずれか。
- 無効ファイルがあればエラーメッセージを表示する。
- 変換ボタンを有効にする。

変換中:

- 変換ボタンを無効にする。
- ファイル単位の状態を更新する。
- ZIPダウンロードボタンは非表示または無効にする。

変換後:

- 成功件数と失敗件数を表示する。
- 1件以上成功していればZIPダウンロードボタンを有効にする。
- 失敗したファイルには理由を表示する。

## 変換フロー

1. ユーザがファイルを選択またはドロップする。
2. Shell層がFile APIからファイルを受け取る。
3. Zodで境界バリデーションを行い、`HeicFile`へ変換する。
4. UI状態へ変換ジョブを追加する。
5. ユーザが出力形式を選ぶ。
6. ユーザが変換ボタンを押す。
7. Shell層が`File.arrayBuffer()`でファイルを読み込む。
8. Core層が`ArrayBuffer`と出力形式を受け取り、`ConvertedBlob`を返す。
9. Shell層が成功結果をUI状態へ反映する。
10. Shell層が成功結果を`jszip`へ渡してZIP Blobを作る。
11. ユーザがZIPをダウンロードする。

## 設計方針

Functional Core:

- HEICデコード、Canvasエンコード、ファイル名生成、ZIPエントリ名生成を小さな関数に分ける。
- 入力と出力の型を明示する。
- 変換結果は`ConvertedBlob` branded typeとして扱う。

Imperative Shell:

- File API、ドラッグ&ドロップ、DOMイベント、URL生成、ダウンロード実行を担当する。
- UI状態管理はReactコンポーネント側で行う。
- ファイル内容をネットワークへ送る処理は持たない。

型:

- `HeicFile`: バリデーション済みHEIC/HEIFファイル。
- `OutputFormat`: `jpg | png | webp`。
- `ConvertedBlob`: 変換済み画像Blob。
- `ConversionJob`: UI上のファイル単位ジョブ。
- `ConversionStatus`: `pending | converting | done | failed`。

## エラー仕様

想定エラー:

- HEIC/HEIFではないファイル。
- `libheif-js`のデコード失敗。
- Canvas作成または描画失敗。
- ブラウザが指定形式の`toBlob`に失敗。
- ZIP生成失敗。

表示方針:

- ファイル単位の失敗は一覧に残して理由を表示する。
- 一部成功していればZIPダウンロードは可能にする。
- 変換全体を中断する致命的エラーは画面上部に表示する。

## プライバシーと制約

- ファイルはブラウザのメモリ内でのみ処理する。
- サーバ送信、外部API送信、アナリティクス送信は実装しない。
- WASMと依存ライブラリはビルド成果物として配信される前提。
- 大量または巨大なファイルではブラウザのメモリ制限に依存する。

## テスト方針

Vitestで確認する項目:

- HEIC/HEIFファイル判定。
- 出力MIMEタイプと拡張子の対応。
- 出力ファイル名生成。
- ZIP内ファイル名の重複回避。
- 変換関数の異常系で失敗理由を返すこと。

手動確認する項目:

- 実HEICファイルのドラッグ&ドロップ。
- 複数ファイル変換。
- JPG、PNG、WebPそれぞれの変換。
- ZIPダウンロードと展開。
- ネットワークタブでファイルアップロードが発生しないこと。

## 受け入れ条件

- `npm run build`が成功する。
- `npm test`が成功する。
- ブラウザでファイル選択とドラッグ&ドロップが動く。
- 複数HEICファイルを選択して変換できる。
- JPG、PNG、WebPのいずれかを選んでZIPを生成できる。
- 変換対象の画像ファイルがサーバへ送信されない。

## 実装前の未確定事項

- アプリ名は仮に`heic-flip`とする。
- JPG/WebP品質はMVPでは固定値`0.92`とする。
- ファイルサイズ上限はMVPでは設けない。
